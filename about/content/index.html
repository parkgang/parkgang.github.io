아래의 것을 수행했어요.

  • 회계 도메인을 이해하고 이에 따른 기능 구현
  • 메인 엔지니어로써 동료들의 코드 리뷰
  • 제품 형태를 이해하고 tRPC , react-hook-form 기술 주도적으로 도입
    이외, TDD , DDD 도 시도

  • 왜, KISS(Keep it small and simple) 가 중요한지
  • Client-Server-Model에 대한 Insight가 깊어지고 제품 설계 능력에 대해서 큰 시야 경험
  • E2E Test와 같은 Test의 모범에 대해서 배우고 경험
  • 도메인 지식의 중요성과 도메인 지식을 다루는 애플리케이션 개발 경험 및 얕은 회계 도메인 획득

현대백화점
2021-07 ~ 2022-01​

Microsoft 365를 도입하면서 아쉬운 부분을 해결하기 위해 개발을 진행하게 되었습니다.
MS Teams Third Party 웹앱을 개발하였습니다.

  • Typescript
  • React
  • recoil
  • react-query
  • styled-components
  • fluentui
  • fluentui react-northstar
  • MUI
  • i18next
  • C#
  • ASP.NET
  • Entity Framework Core
  • Serilog
  • Microsoft Graph
  • Azure Application Insights
  • IIS
  • 2중화 환경

개발된 제품은 아래와 같아요.

  • 팀즈 보안 파일 뷰어 (DRM 및 MIP으로 암호화된 파일을 복호화 해서 파일 미리 보기를 제공)
    • 스켈레톤 UI, ProgressBar를 이용하여 Non-Blocking 되도록 UX 대폭 개선
      • 기존에는 클릭 Action마다 화면 전체에 로더를 표시하여 오랜 시간 동안 사용자가 다른 Action을 할 수 없던 것을 화면 최상단에 ProgressBar를 표시하여 요청이 실행 중임을 나타내고 그동안 Non-Blocking이 되어 다른 Action을 추가적으로 진행할 수 있도록 개선
      • 베타 테스트 결과 실제 API 속도 개선이 이뤄진 것은 아니지만 UX으로 인하여 속도가 매우 빨라졌다는 경험을 전달할 수 있었음
    • 스파게티 React 코드 분석 후 최적화
      • 여러 군데에서 queryString을 조회하는 코드를 단일 책임 원칙을 지키도록 Custom Hook으로 제작
      • 불필요한 Prop Drilling을 Global Status으로 전환하여 중첩 구조 최적화
      • 불변성을 지키지 않는 state code 픽스
  • 팀즈 바로 가기 (외부 시스템에 편하게 접속할 수 있는 바로 가기 링크 제공)
    • 링크 클릭 시 Token이 발급되고 Token을 검증할 수 있는 API를 제공
    • 여러 엔지니어가 거쳐간 제품으로 스트럭처가 일정하지 않아 유지 보수성이 떨어지는 제품이었는데 해당 프로젝트에서 진행된 React 제품들과 유사하게 리펙토링을 통해 후임 엔지니어의 유지 보수성 개선
    • API 인증 KEY와 요청받을 수 있는 IP를 제한하여 보안적으로 더 단단한 API를 설계해야 한다는 것을 알 수 있었음
  • 팀즈 조직도 (로그인한 사용자 권한에 맞게 조직도 정보를 조회/탐색/검색 기능 제공)
    • 하이어라키 구조를 Tree 형태로 표현하도록 UI 제작
    • 정보 클릭 시 Table 구조로 사용자 정보를 보여주도록 제작
    • 검색 시 사용자 정보를 하이라이팅 하여 보여주며 PC에서는 more으로, Mobile에서는 무한 스크롤로 탐색 제공
    • Mobile에서 그룹 채팅 초대 UI를 위해 Drawer UI PoC 후 적용
    • 사용자 사진 동기화를 위하여 ASP.NET Core 백그라운드 작업을 이용하여 동기화 코드 제작
    • 사용자 사진 HTTP 캐시 설계
    • 첫 개발 당시 느슨하게 잡힌 타입을 강하게 잡아서 유지 보수성을 개선

교보생명
2021-03 ~ 2021-07​

Microsoft 365를 도입하면서 아쉬운 부분을 해결하기 위해 개발을 진행하게 되었습니다.
MS Teams Third Party 웹앱을 개발하였습니다.

  • Typescript
  • React
  • recoil
  • react-query
  • styled-components
  • fluentui react-northstar
  • C#
  • ASP.NET
  • Serilog
  • Microsoft Graph
  • Azure Application Insights
  • DRM
  • IIS

개발된 제품은 아래와 같아요.

  • 팀즈 전자결재 (고객 측 API를 통해 Teams에서 전자결재를 처리할 수 있도록 제공)
    • 외부의 API 직접 소비해 보면서 문제가 없는지 검증하고 소비가 불편한 interface도 경험하면서 어떻게 설계하는 것이 올바른가 생각할 수 있었음
    • Client Side, Server Side 로그를 직접 심어보면서 어떻게 남겨야 추적성이 좋을까? 필요한 정보를 추출할 수 있을까? 로그 설계의 어려움을 경험
  • 팀즈 다운로드 (직접 다운로드를 금지한 오피스 클라우드의 파일에서 MIP를 해제하고 DRM을 걸어 Teams 앱 내부에서 다운로드할 수 있도록 제공)
    • DRM과 같이 보호된 솔루션을 클라우드 환경의 다른 SaaS(M365, Teams)에서 다운로드할 수 있는 설계를 이해하고 구현한 경험
    • 자연스럽게 MIP(Microsoft Information Protection)도 알게 되며 파일 Level, OS Level, 등 여러 종류의 암호화가 있다는 것을 알 수 있었음
    • 이 과정에서 DRM 모듈을 사용해 보며 어떤 암호화 기능이 있는지 알 수 있었음
    • 고객이 DRM을 왜 사용하는지, 해당 제품을 도입해서 얻고 싶은 가치가 무엇인지, 경험할 수 있었음
